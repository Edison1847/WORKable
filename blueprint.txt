Data  flows  from  L1  (input)  →  L2  (store)  →  L3  (thinking)  →  L4  (decision)  →  L5  (interaction).  The  continuous
learning loop feeds completed action outcomes back to L1, making the system progressively smarter over time.

L4 — Decision & ActionThe judgment — from insight to recommendationRule-based recommendations · Simulation engine · Nudge mechanism · Action state bridge (Pending→Completed)
L3 — Thinking SystemThe brain — four tiers of progressive intelligenceTier A: Signal prep · Tier B: 13 direct analytics · Tier C: 30+ deep reveals · Tier D: HSI · Culture · Financial impact
L2 — Data StoreThe memory — structured storage with separation enforcementTime-series · Graph · Relational · Longitudinal audit · Separation enforcement · Semantic embedding

SECTION 03
Layer 1 — Input System
Unit  of  analysis:  each  individual  work  activity  —  not  the  person  as  a  whole.  Worker  and  manager  answer
independently. The gap between their views is the primary data source.
[DEMO] Progressive Disclosure Onboarding
Questions  released  in  stages  based  on  completion  state  and  role  context.  Prevents  cognitive  overload,  improves
data quality. Unlocks deeper tiers only when prior tiers are complete.
[DEMO] First Pulse — 3-Question Pre-Diagnostic
60-second LLM-powered pre-diagnostic before full onboarding. Captures the three most urgent signals. Produces an
immediate preliminary read on the organisation. AI type: LLM with structured context.
[DEMO] Emotional Tone Layer on Free Text
NLP classifier (sentiment + urgency) on Q9 (blockers), Q11 (improvements), Q14 (recovery). Detects fear patterns,
resignation language, and urgency spikes. AI type: NLP classifier.
[DEMO] Consistency Fingerprinting
Detects  statistical  anomalies  in  worker  response  patterns  within  a  single  audit  cycle.  Flags  implausibly  uniform  or
internally inconsistent responses. AI type: ML anomaly detection.
[DEMO] Worker Trust Score
Composite response reliability measure — consistency fingerprint + audit stability + emotional tone. Used for internal
signal weighting. Never shown to the worker. AI type: ML composite model.
[DEMO] Temporal Drift Signal
Compares each worker's activity description embedding across consecutive audit cycles. Material divergence flags
role drift. Powers HC-RET-002 Temporal Drift Index. AI type: embedding + cosine similarity.

SECTION 04
Layer 2 — Data Store
Layer 2 transforms, structures, and stores raw inputs — and enforces the rules that make the data trustworthy.
Separation enforcement is the architectural trust guarantee.
Separation Enforcement — The Trust Guarantee
Worker  and  manager  answers  are  stored  in  completely  separate  tables  with  a  database-level  rule  preventing
any direct join without passing through the comparison engine. If workers believed their manager could see their
raw responses, they would answer strategically and all diagnostic value would collapse.
[DEMO] Separation Enforcement — Immutable Audit Log
Every access to the separation layer is cryptographically logged. Attempts to join worker and manager tables outside
the controlled comparison engine produce an alert and are written to an append-only tamper-evident audit log. This
is a GDPR, POPIA, and PDPA compliance mechanism.
STOREWHAT IT HOLDSPOWERS
Time-seriesEvery numerical signal per person per activity per audit dateTrend detection, Chronic Overload Loop, Future State Engine
Graph storeRelationships between entities — people, roles, teams, activitiesOrg twin layout, hidden network detection, Shadow Org View
RelationalStructured reference data — worker profiles, onboarding, salary bandsFinancial model, manager effectiveness, all reporting
Longitudinal auditAppend-only complete org-wide snapshots frozen at a point in timeVoice Suppression detection, Org Memory System, benchmarking
Semantic Embedding Layer — Hidden Work Detection
Converts free-text activity descriptions into numerical vectors. Similarity score below threshold (default 0.4) flags
the activity as hidden work. AI type: embedding model.
[DEMO] Organisational Dark Matter
Hidden work activities are clustered spatially. When multiple workers in the same department independently report
activities  not  appearing  in  any  manager's  expected  list,  these  form  a  Dark  Matter  cluster  —  work  the  organisation
funds but does not officially recognise or manage.

[DEMO] Cross-Person Reveal Clustering
When  the  same  reveal  fires  for  multiple  people  in  the  same  team  +  timeframe,  it  becomes  a  cluster.  Cluster  =
systemic pattern, not individual issue. AI type: spatial/temporal association rule mining.
[DEMO] Reveal Interaction Map (Syndrome Pairs)
Identifies co-occurrence patterns in reveals. High Effort–Low Meaning + Support Void = burnout syndrome. Labels
them as named syndromes with combined financial impact and compound action. AI type: association rule mining.
[DEMO] Organisational Immune System
Detects self-opposition patterns — reveals actively working against each other. A Reallocate nudge accepted while
Target Unrealism is also active = the interventions cancel each other. AI type: constraint satisfaction model.
[DEMO] Organisational Entropy Score (OES) — HC-ENT-001
Weighted variance across all Tier B signals for all people in all activities. High entropy = the organisation is pulling in
many directions simultaneously. Entropy rising over consecutive cycles is the earliest leading indicator of a human
system crisis. AI type: statistical variance model — deterministic by design.

[DEMO] Action Sequencing — Dependency Graph + Prescribed Order
Actions  are  not  independent.  Clarify  must  precede  Coach.  Re-scope  must  precede  Reallocate.  Every  Action
Command Centre output includes a prescribed execution order. AI type: constraint-satisfaction dependency graph.
[DEMO] Inaction Cost Clock — Daily $ Accumulator
Every pending nudge carries a daily financial counter. The clock starts the moment the nudge fires and increments
daily  by  the  pro-rated  financial  impact.  Current  total:  $71,000/month.  AI  type:  deterministic  arithmetic  —  no  AI  by
design (auditability requirement).
[DEMO] Pre-Mortem Engine
Given current reveal trajectories, projects the most probable adverse outcome sequence if no action is taken — and
identifies the minimum intervention set that breaks the chain at the earliest point. AI type: LLM with pre-fetched signal
context. Upgrades to agent when signal history exceeds context window.
[DEMO] Intervention Genome — Multi-Dimensional Outcome Prediction
The  core  long-term  moat.  Every  completed  action  stored  with:  reveal  type,  action  type,  org  context,  time  to
completion, signal delta at next audit. Powers prediction: given this reveal in this org context, which action produces
the best outcome? AI type: ML outcome prediction model — trained continuously.
[DEMO] What-If Studio
Open-ended  NL  scenario  simulation.  Manager  types  any  scenario  in  plain  English.  An  agent  translates  it  into
simulation parameters, runs multiple variants, and returns ranked outcome comparison. AI type: genuine agentic —
open-ended NL → simulation calls → multi-step reasoning → explanation.

Organisational Biological Age (OBA) — HC-BIO-001
[FUTURE] This is a future concept — not in current build scope
OBA requires minimum 4 audit cycles per organisation. The formula, markers, and methodology are defined here for
the blueprint record. Implementation begins in Phase 2.
Grounded  in  Horvath's  epigenetic  clock  (2013),  cellular  senescence  research,  VO2  max  measurement,  and
telomere biology. Chronological age tells you how long an organisation has existed. Biological age tells you how
well its systems are actually functioning relative to that stage of existence. The gap between the two is the most
actionable number in organisational health.
